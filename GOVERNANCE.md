# Governance

## Maintainers

Polis Protocol is maintained by **DeegaLabs**:

- **Daniel Gorgonha** — Technical lead, architecture, smart contracts
- **Dayane Gorgonha** — Product, design, community

## Decision making

- **Protocol changes** require consensus from both maintainers
- **Bug fixes** can be merged by either maintainer after review
- **New dependencies** require explicit approval
- **Breaking changes** require a migration guide and changeset

## Release process

1. Changes land on `main` via squash merge
2. Changesets accumulate in a release PR
3. Release PR is merged by a maintainer
4. Packages are published to npm automatically via CI

## Communication

- Issues and PRs on GitHub
- Major decisions documented in ADRs (Architecture Decision Records)
